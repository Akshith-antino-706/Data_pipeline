GET
Get template content templates

/apis/v1/services/account/templates/content/?c={account}&template_id={id}

Returns the rendered HTML content of a specific template.

⌄

Query parameters

crequiredstringAccount identifier (e.g. rayna)
template_idrequiredintegerID of the template to retrieve (e.g. 1236)

Example request

GET https://ser1.chathead.io/apis/v1/services/account/templates/content/?c=rayna&template_id=1236

Response

ⓘ Returns plain HTML, not a JSON envelope. The response body is raw template markup ready to render directly in a browser or email client.
Content-Type: text/html <div class="template-body"> <p>Hello {{customer_name}},</p> <p>Your booking is confirmed.</p> </div>

The returned HTML may contain placeholder variables (e.g. {{customer_name}}) that should be substituted server-side before rendering to end users.

Broadcast
POST
Upload data file broadcast

/apis/v1/services/broadcast/data/add/

Uploads a recipient data file for use in a broadcast. Only .data files are accepted.

⌄

Request

MethodPOST
Content-Typemultipart/form-data

Form fields

Filedatarequired file The data file to upload. Must have a .data extension. Content-Type must be application/octet-stream.
clientrequired string Account identifier (e.g. rayna)

File constraint

△ Only files with the .data extension are accepted. Uploads with any other extension will be rejected by the server.

Example request

cURL
JavaScript
PHP
curl -X POST https://ser1.chathead.io/apis/v1/services/broadcast/data/add/ \ -F "Filedata=@sample.data;type=application/octet-stream" \ -F "client=rayna"
const formData = new FormData(); formData.append('Filedata', file); // .data extension required formData.append('client', 'rayna'); const res = await fetch( 'https://ser1.chathead.io/apis/v1/services/broadcast/data/add/', { method: 'POST', body: formData } ); const result = await res.json();
$curl = curl_init(); curl_setopt_array($curl, [ CURLOPT_URL => 'https://ser1.chathead.io/apis/v1/services/broadcast/data/add/', CURLOPT_POST => true, CURLOPT_RETURNTRANSFER => true, CURLOPT_POSTFIELDS => [ 'Filedata' => new CURLFile('/path/to/sample.data', 'application/octet-stream', 'sample.data'), 'client' => 'rayna', ], ]); $response = curl_exec($curl); curl_close($curl);

GET
Add broadcast broadcast

/apis/v1/services/broadcast/add/?c={account}&name={name}&channel={id}&subject={subject}&data_file={file}&template_id={id}&send_time={datetime}

Creates and schedules a new broadcast using an uploaded data file and a message template.

⌄

Query parameters

crequiredstringAccount identifier (e.g. rayna)
namerequiredstringDisplay name for the broadcast — URL-encoded (e.g. test%20from%20api)
channelrequiredintegerChannel ID to send through (e.g. 140)
subjectrequiredstringBroadcast subject line — URL-encoded
data_filerequiredstringFilename of the previously uploaded .data file (e.g. sample.data)
template_idrequiredintegerID of the message template to use (e.g. 1144)
send_timerequiredstringScheduled datetime in YYYY-MM-DD HH:MM:SS format, URL-encoded (e.g. 2026-05-28%2005:57:00)

Typical flow

→ Step 1 — Upload the recipient data file via POST /broadcast/data/add/ and note the filename.
→ Step 2 — Pass that filename as data_file in this endpoint to create and schedule the broadcast.

Example request

GET https://ser1.chathead.io/apis/v1/services/broadcast/add/?c=rayna&name=test%20from%20api&channel=140&subject=hey%20there&data_file=sample.data&template_id=1144&send_time=2026-05-28%2005:57:00

Response — 200 success

{ "status": "success", "msg": "Broadcast Added!", "data": { "broadcast_id": 212 } }

Response fields — data

broadcast_idintegerUnique identifier of the newly created broadcast. Use this to reference or track the broadcast in subsequent calls.
