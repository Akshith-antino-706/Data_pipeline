Rayna Tours — GTM Webhook Setup

15 individual Custom HTML tags for GTM — one per event, each with its own trigger. Forwards dataLayer events to our segmentation pipeline in real-time.

1. What This Does

Your GTM already fires these 15 events. This webhook tag intercepts them automatically and sends a copy to our database for customer segmentation. No changes needed to your existing tags or triggers.

Event NameWhen It FiresTypeWhat We Track
view_itemProduct/tour page viewedECOMMERCEitem_id, item_name, item_category, price
add_to_cartAdded to cartECOMMERCEitems[], value, name, email_id, city, selected_date, image_url
check_availability_clickAvailability check clickedECOMMERCEitem context, page URL
event_dateDate selected for a tourECOMMERCEselected_date, item context
begin_checkoutCheckout startedCONVERSIONitems[], value, coupon
add_payment_infoPayment details enteredCONVERSIONpayment_type, value
purchaseBooking confirmedCONVERSIONtransaction_id, value, items[], name, email_id, contact_number, city
payment_failedPayment errorCONVERSIONerror context, cart value
generate_leadLead form submittedCONVERSIONlead type, product interest
leadEnquiry submittedCONVERSIONlead type, source
click_whatsappWhatsApp CTA clickedENGAGEMENTclick location, product context
click_callPhone CTA clickedENGAGEMENTclick location, page URL
click_emailEmail CTA clickedENGAGEMENTclick location, page URL
shareShare button clickedENGAGEMENTshare method, product context
add_to_wishlistWishlist actionENGAGEMENTitem_id, item_name

2. Webhook Endpoint

POST https://qh6hjtm8-3001.inc1.devtunnels.ms/api/v3/gtm/events
Note: Replace https://localhost:3443 with the production API URL once deployed.

3. Tags — Copy & Paste into GTM

For each event below:

1 Go to GTM → Tags → New → Custom HTML

2 Paste the script for that event

3 Set Trigger: Custom Event → event name as shown

4 Save & Publish

Each tag is fully self-contained. No shared dependencies. Each fires independently on its own trigger.

1 In GTM, click Preview to enter debug mode

2 Browse the Rayna Tours website — trigger a few events (view a product, click WhatsApp, etc.)

3 Check our API to see if events arrived:

curl https://localhost:3443/api/v3/gtm/analytics

4 You should see event counts for each event type

Expected result: Each event you trigger on the website should appear in the API response within 1-2 seconds. If you see "event_count": 1 for the events you triggered, the pipeline is working.

5. Architecture


  User browses raynatours.com
         │
         ▼
    GTM dataLayer.push()  ← your existing tags (no changes needed)
         │
         ├──► GA4 → BigQuery (30min delay, fallback only)
         │
         └──► This Webhook Tag
                   │
                   ▼ (real-time, <1 sec)
              POST /api/v3/gtm/events
                   │
                   ▼
              PostgreSQL (gtm_events table)
                   │
                   ▼
              Segmentation Engine (28 customer segments)
                   │
                   ▼
              Personalized campaigns (Email / WhatsApp / SMS)

6. FAQ

Will this slow down the website?

No. It uses navigator.sendBeacon() which is non-blocking and works even during page navigation. Zero impact on page load speed.

Do I need to change my existing GTM tags?

No. This tag listens to the dataLayer passively. Your existing GA4 tags, MoEngage tags, and all other tags continue working exactly as before.

What if the API is down?

Events silently fail with no console errors or user impact. BigQuery serves as a fallback — we can backfill any missed events from there.

What about user privacy / CORS?
